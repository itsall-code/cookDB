# Cook-DB Frontend Skills · Qt Style UI

## 设计角色
以高级 UI 设计师视角维护前端界面，优先保证数据工具的可读性、确定性和低误操作率。

## Qt 风格准则
- **窗口感**：页面应像桌面工具窗口，使用顶部标题栏、浅灰工作区、分组面板和清晰边框。
- **控件质感**：按钮、输入框、选择器采用轻微渐变、1px 描边、内阴影与紧凑尺寸，强调原生桌面控件反馈。
- **信息层级**：每个操作区域以 `QGroupBox` 式分组呈现，标题短、说明轻、危险操作用独立警示面板。
- **状态优先**：连接状态、日志与确认信息始终保持可见；成功、警告、错误颜色要克制且符合工具软件语义。
- **密度控制**：保留桌面工具的高信息密度，但使用 8px 栅格、统一行高与一致的控件间距避免拥挤。

## 实施约束
- 不改变已有 DOM id，避免影响 `popup.js`、`options.js` 与 `mysql.js` 的事件绑定。
- 优先通过 CSS design tokens 调整视觉系统；需要新增说明文案时保持中文工具语境。
- 前端改动需覆盖 Popup、Options 与 MySQL Workbench 三个入口的共同组件。
